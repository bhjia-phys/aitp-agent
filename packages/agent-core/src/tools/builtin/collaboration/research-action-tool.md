Inspect and record Hakimi research actions.

Use this tool to list available semantic research actions, render the primitive tool plan for an action, ask the scheduler which action should handle open obligations next, open/switch/list/close active WorkFrames, compile/load bounded ContextPacks for a WorkFrame, inspect the attached DomainPack manifest, list/load WorkFrame-scoped evidence refs, inspect AITP runtime payload profile metadata, inspect/search AITP curated heuristic RAG context, draft read-only curated RAG promotion plans and prefilled write-bridge call drafts, inspect handoff guard remediation taxonomy metadata, run deterministic in-process research executors, execute a configured AITP write bridge, or record the outcome of a semantic research action. This tool can execute only built-in graph queries, formalization-blueprint export, registered benchmark adapters, scoped ResearchLedger evidence inspection, read-only handoff remediation taxonomy inspection, read-only AITP runtime payload profile inspection, read-only AITP curated RAG inspection/search/draft planning, and explicitly configured AITP write-bridge operations. It does not execute arbitrary shell, git, web, MCP, or HPC work by itself. Primitive tool execution should remain attributed separately and then recorded back through the action result.

Curated RAG handoffs remain draft-only until a separate explicit `execute_aitp_write_bridge` call provides top-level `aitp_operation` and `aitp_payload`. When an optional `aitp_handoff` fails guard verification, the result includes `handoff_guard_failure` and a `remediation_summary` with stable repair steps: `copy_missing_handoff_field_from_draft`, `align_explicit_execute_args_with_handoff_tool_call`, `redraft_handoff_or_restore_hash_input`, `redraft_or_resolve_blocking_diagnostics`, or `inspect_handoff_guard_failure`. `inspect_handoff_guard_remediation_taxonomy` returns the same failure-code to repair-step map as read-only metadata. These are repair hints only; they never mutate the handoff, call the bridge, record evidence, validate, satisfy final gates, or update claim trust.
